🧭 Introduzione e Protagonisti

Creare un server con un clic è facile, ma farlo creare in automatico da un pezzo di codice è il vero "superpotere" del Cloud. In questo laboratorio, trasformeremo un concetto complesso (l'infrastruttura come codice) in un'esperienza visiva e pratica, guidandoti in ogni singolo clic.

Prima di iniziare, è utile capire quali "mattoncini" AWS stiamo per incastrare tra loro. Ecco una sintesi dei servizi coinvolti:

Servizio AWS Il suo ruolo nel mondo reale Cosa farà nel nostro laboratorio
VPC & Subnet Il quartiere e la via di casa Fornirà l'indirizzo di rete al nostro nuovo server
Security Group Il "buttafuori" all'ingresso Deciderà chi può connettersi al server (es. tramite SSH)
IAM Role Il badge di sicurezza aziendale Darà alla nostra funzione Lambda il permesso di creare server
AWS Lambda Il robot esecutore Leggerà il nostro codice Python e lo eseguirà su richiesta
Amazon EC2 Il computer fisico/virtuale Sarà il risultato finale: un server funzionante e pronto all'uso
👋 Benvenuto nel Laboratorio! Se non hai mai programmato o usato AWS, sei nel posto giusto. L'obiettivo di oggi è scrivere un piccolo script che, premendo un pulsante, "ordinerà" ad AWS di accendere un nuovo computer virtuale per noi.

Step 1: Trovare l'indirizzo e il "Buttafuori" 🏠

Per creare un server (EC2), AWS ha bisogno di sapere due cose: in quale "via" della rete metterlo (Subnet) e chi può entrarci (Security Group).

A. Troviamo la Subnet (La Via)

  • Accedi alla tua console AWS. In alto a sinistra, nella barra di ricerca, scrivi VPC e cliccaci sopra.
  • Nel menu a sinistra, clicca su Subnets.
  • Vedrai una lista di elementi. Scegline uno qualsiasi e copia il suo Subnet ID (assomiglia a subnet-0abcd1234...). Incollalo in un blocco note sul tuo computer: ci servirà dopo!

B. Creiamo il Security Group (Il Buttafuori)

  • Ora cerca EC2 nella barra di ricerca in alto e cliccaci.
  • Nel menu a sinistra (scorri un po' in basso), clicca su Security Groups.
  • Clicca sul pulsante arancione Create security group in alto a destra.
  • Security group name: scrivi Regole-Mio-Server.
  • Description: scrivi Permette l'accesso al server.
  • Scorri giù fino a Inbound rules (Regole in entrata) e clicca su Add rule.
  • Sotto Type scegli SSH. Sotto Source scegli Anywhere-IPv4.
  • Scorri in fondo e clicca su Create security group.
  • Nella pagina che si apre, copia il Security group ID (assomiglia a sg-0987xyz...) e salvalo nel tuo blocco note.

Step 2: Creare il Badge di Sicurezza (IAM Role) 🪪

In AWS, i servizi non si fidano l'uno dell'altro. La nostra funzione Lambda (il robot) non ha il permesso di creare server (EC2). Dobbiamo crearle un "badge" con i permessi giusti.

  • Cerca IAM nella barra di ricerca in alto e clicca.
  • A sinistra, clicca su Roles (Ruoli) e poi sul pulsante blu Create role.
  • Seleziona AWS service, poi scegli Lambda dalla lista e clicca Next in basso a destra.
  • Nella barra di ricerca delle policy, scrivi AWSLambdaBasicExecutionRole, premi Invio e metti la spunta nel quadratino accanto al risultato. Clicca Next.
  • Role name: scrivi Badge-Per-Creare-Server. Scorri in fondo e clicca Create role.
  • Ora, nella lista dei ruoli, cerca il tuo Badge-Per-Creare-Server e cliccaci sopra.
  • A destra, clicca su Add permissions e poi su Create inline policy.
  • Clicca sulla linguetta JSON (in alto a destra nell'editor nero) e cancella tutto quello che c'è scritto. Incolla questo codice esatto:
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "ec2:RunInstances",
                "ec2:CreateTags"
            ],
            "Resource": "*"
        }
    ]
}
  • Clicca su Next. Dai il nome Permessi-EC2 e clicca su Create policy. Il badge è pronto!

Step 3: Scrivere il Codice Python nella Lambda 🧠

Ora uniamo i pezzi. Creeremo la funzione Lambda e le daremo le istruzioni in Python.

  • Cerca Lambda nella barra di ricerca in alto e clicca.
  • Clicca sul pulsante arancione Create function.
  • Function name: scrivi CreaMioServer.
  • Runtime: scegli Python 3.12 (o una versione più recente).
  • Apri la tendina Change default execution role. Scegli Use an existing role e seleziona il tuo Badge-Per-Creare-Server.
  • Clicca Create function in basso a destra.

Il Codice (Non spaventarti, è tutto spiegato!)

Scorri la pagina fino a trovare l'editor di codice con lo sfondo nero. Cancella tutto il codice presente e incolla quello qui sotto. Importante: Sostituisci i valori in arancione con quelli che hai salvato nel blocco note!

# Boto3 è il "telecomando" che Python usa per parlare con AWS
import boto3 

# Questa è la funzione principale che AWS avvia quando premiamo "Test"
def lambda_handler(event, context):
    
    # 1. Prendiamo in mano il telecomando e diciamo che vogliamo usare EC2
    # Nota: Cambia 'eu-west-1' se ti trovi in un'altra regione (es. eu-south-1 per Milano)
    telecomando_ec2 = boto3.client('ec2', region_name='eu-west-1')
    
    # 2. Prepariamo i dati del nostro server (SOSTITUISCI I DATI QUI SOTTO!)
    # AMI_ID è il sistema operativo. Questo è Amazon Linux 2023 (gratuito).
    AMI_ID = 'ami-0c1c30571d2dae5c9' 
    TIPO_SERVER = 't2.micro' # Un server piccolo e gratuito
    
    # INCOLLA QUI I TUOI ID SALVATI NEL BLOCCO NOTE:
    MIA_SUBNET = 'subnet-INCOLLA_QUI_IL_TUO_ID'
    MIO_SECURITY_GROUP = 'sg-INCOLLA_QUI_IL_TUO_ID'
    
    # 3. Diamo l'ordine ad AWS di creare il server!
    try:
        risposta = telecomando_ec2.run_instances(
            ImageId=AMI_ID,
            InstanceType=TIPO_SERVER,
            MinCount=1, # Vogliamo minimo 1 server
            MaxCount=1, # Vogliamo massimo 1 server
            SubnetId=MIA_SUBNET,
            SecurityGroupIds=[MIO_SECURITY_GROUP],
            TagSpecifications=[
                {
                    'ResourceType': 'instance',
                    'Tags': [{'Key': 'Name', 'Value': 'Server-Automatico-Python'}]
                }
            ]
        )
        
        # Se arriviamo qui, ha funzionato! Estraiamo l'ID del nuovo server.
        id_nuovo_server = risposta['Instances'][0]['InstanceId']
        print(f"Evviva! Server creato con successo. ID: {id_nuovo_server}")
        
        return "Missione compiuta!"
        
    except Exception as errore:
        # Se qualcosa va storto, stampiamo l'errore per capire il problema
        print(f"Ops, c'è stato un problema: {errore}")
        return "Errore durante la creazione."
  • Dopo aver modificato i dati, clicca sul pulsante Deploy (sopra il codice) per salvare le modifiche.

Step 4: Il Grande Test e la Pulizia Finale 🎯

È il momento della verità. Facciamo partire il nostro codice.

  • Sempre nella pagina della tua Lambda, clicca sul pulsante azzurro Test.
  • Ti chiederà di creare un evento di test. Scrivi MioTest come Event name, scorri in basso e clicca su Save.
  • Clicca di nuovo sul pulsante azzurro Test.
  • Se vedi un riquadro verde con scritto "Missione compiuta!", congratulazioni! Hai appena programmato il Cloud.
  • Per verificare con i tuoi occhi, cerca EC2 nella barra in alto, clicca su Instances (Istanze) a sinistra e vedrai il tuo nuovo Server-Automatico-Python in fase di avvio!
⚠️ FASE CRITICA: Non dimenticare di spegnere la luce!
Il cloud si paga a consumo. Ora che hai visto che funziona, seleziona il tuo nuovo server nella console EC2, clicca in alto a destra su Instance state e scegli Terminate instance. Questo distruggerà il server e bloccherà qualsiasi costo futuro.

💡 Consigli per l'Insegnamento e Best Practices

Se stai seguendo o erogando questo laboratorio, tieni a mente questi punti chiave per evitare gli ostacoli più comuni tra i principianti:

  • La questione della "Region": I principianti spesso si perdono perché creano la Lambda a Francoforte e cercano il server a Milano. Assicurati di controllare in alto a destra nella console AWS in quale regione ti trovi (es. eu-west-1 per Irlanda) e fai coincidere questo valore nel codice Python.
  • L'AMI ID: L'ID del sistema operativo (ami-...) cambia frequentemente e varia da regione a regione. Se l'esecuzione fallisce con un errore di "Invalid AMI", puoi copiare un AMI ID aggiornato direttamente dalla schermata di lancio manuale di EC2.
  • L'importanza del "Terminate": L'avviso nel box giallo è vitale. I principianti temono sempre di ricevere fatture altissime da AWS; distruggere le risorse a fine laboratorio costruisce fiducia nel processo di apprendimento.
