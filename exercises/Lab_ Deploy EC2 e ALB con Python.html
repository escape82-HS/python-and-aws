⚖️ Lab: Deploy di EC2 e ALB con Python

Impara a creare un'architettura ad alta affidabilità: un Application Load Balancer pubblico che distribuisce il traffico a due server web privati, tutto automatizzato con Boto3.

📋 Prerequisiti

Prima di iniziare, assicurati di avere:

  • Credenziali AWS configurate (aws configure) con permessi sufficienti (EC2, ELB).
  • Python 3 e la libreria boto3 installati.
  • Infrastruttura di rete esistente nel tuo account AWS:
    • 1 VPC
    • 2 Public Subnet (connesse a un Internet Gateway)
    • 2 Private Subnet (connesse a un NAT Gateway per scaricare Apache)

1 Raccogli gli ID della tua Rete

Per dire a Python dove creare le risorse, devi recuperare gli ID della tua rete dalla Console AWS (servizio VPC). Annotali, ti serviranno nel prossimo step.

VPC_ID = "vpc-xxxxxxxxxxxxxxxxx"
PUB_SUBNET_1 = "subnet-xxxxxxxxxxxxxxxxx" (es. us-east-1a)
PUB_SUBNET_2 = "subnet-yyyyyyyyyyyyyyyyy" (es. us-east-1b)
PRIV_SUBNET_1 = "subnet-zzzzzzzzzzzzzzzzz" (es. us-east-1a)
PRIV_SUBNET_2 = "subnet-wwwwwwwwwwwwwwwww" (es. us-east-1b)

2 Lo Script di Deployment

Crea un file chiamato deploy_alb_ec2.py. Questo script farà molte cose: creerà i Security Group, lancerà le istanze installando Apache tramite User Data, e configurerà il Load Balancer.

import boto3
import time

# Inizializza i client Boto3
ec2 = boto3.client('ec2', region_name='us-east-1')
elbv2 = boto3.client('elbv2', region_name='us-east-1')

# ==========================================
# 🛑 INSERISCI QUI I TUOI ID
# ==========================================
VPC_ID = 'vpc-INSERISCI_QUI'
PUB_SUBNETS = ['subnet-PUB1', 'subnet-PUB2']
PRIV_SUBNETS = ['subnet-PRIV1', 'subnet-PRIV2']
AMI_ID = 'ami-0c7217cdde317cfec' # Ubuntu 22.04 in us-east-1
# ==========================================

print("🚀 Inizio del deployment dell'infrastruttura...")

# 1. Creazione Security Group per l'ALB (Aperto al mondo su porta 80)
alb_sg = ec2.create_security_group(GroupName='ALB-SG-Lab', Description='SG per Application Load Balancer', VpcId=VPC_ID)
alb_sg_id = alb_sg['GroupId']
ec2.authorize_security_group_ingress(
  GroupId=alb_sg_id,
  IpPermissions=[{'IpProtocol': 'tcp', 'FromPort': 80, 'ToPort': 80, 'IpRanges': [{'CidrIp': '0.0.0.0/0'}]}]
)
print(f"✅ Creato ALB Security Group: {alb_sg_id}")

# 2. Creazione Security Group per le EC2 (Accetta traffico SOLO dall'ALB)
ec2_sg = ec2.create_security_group(GroupName='EC2-SG-Lab', Description='SG per Web Servers', VpcId=VPC_ID)
ec2_sg_id = ec2_sg['GroupId']
ec2.authorize_security_group_ingress(
  GroupId=ec2_sg_id,
  IpPermissions=[{'IpProtocol': 'tcp', 'FromPort': 80, 'ToPort': 80, 'UserIdGroupPairs': [{'GroupId': alb_sg_id}]}]
)
print(f"✅ Creato EC2 Security Group: {ec2_sg_id}")

# 3. User Data: Script Bash per installare Apache e creare la pagina web
user_data_script = """#!/bin/bash
apt-get update -y
apt-get install apache2 -y
systemctl start apache2
systemctl enable apache2
echo "<html><head><style>body{font-family:sans-serif; text-align:center; margin-top:50px;}</style></head><body>" > /var/www/html/index.html
echo "<h1>Benvenuto nel Lab AWS di MarbleMind! 🚀</h1>" >> /var/www/html/index.html
echo "<p>Questa pagina e' servita dall'istanza: <strong>$(hostname -f)</strong></p>" >> /var/www/html/index.html
echo "</body></html>" >> /var/www/html/index.html
"""

# 4. Lancio delle due Istanze EC2 nelle Private Subnet
istanze_create = []
for i, subnet in enumerate(PRIV_SUBNETS):
  response = ec2.run_instances(
      ImageId=AMI_ID, InstanceType='t2.micro', MinCount=1, MaxCount=1,
      SecurityGroupIds=[ec2_sg_id], SubnetId=subnet, UserData=user_data_script,
      TagSpecifications=[{'ResourceType': 'instance', 'Tags': [{'Key': 'Name', 'Value': f'WebServer-Lab-{i+1}'}]}]
  )
  istanze_create.append(response['Instances'][0]['InstanceId'])

print(f"⏳ Istanze EC2 in creazione: {istanze_create}. Attesa dell'avvio...")
waiter = ec2.get_waiter('instance_running')
waiter.wait(InstanceIds=istanze_create)
print("✅ Istanze EC2 avviate.")

# 5. Creazione del Target Group
tg_response = elbv2.create_target_group(
  Name='Lab-Target-Group', Protocol='HTTP', Port=80, VpcId=VPC_ID, TargetType='instance'
)
tg_arn = tg_response['TargetGroups'][0]['TargetGroupArn']

# 6. Registrazione delle istanze nel Target Group
targets = [{'Id': instance_id} for instance_id in istanze_create]
elbv2.register_targets(TargetGroupArn=tg_arn, Targets=targets)
print("✅ Target Group creato e istanze registrate.")

# 7. Creazione dell'Application Load Balancer
alb_response = elbv2.create_load_balancer(
  Name='Lab-ALB-Web', Subnets=PUB_SUBNETS, SecurityGroups=[alb_sg_id], Scheme='internet-facing', Type='application'
)
alb_arn = alb_response['LoadBalancers'][0]['LoadBalancerArn']
alb_dns = alb_response['LoadBalancers'][0]['DNSName']

# 8. Creazione del Listener per l'ALB
elbv2.create_listener(
  LoadBalancerArn=alb_arn, Protocol='HTTP', Port=80,
  DefaultActions=[{'Type': 'forward', 'TargetGroupArn': tg_arn}]
)
print("✅ Application Load Balancer e Listener creati.")

print("\n🎉 DEPLOYMENT COMPLETATO! 🎉")
print("=====================================================")
print(f"🌐 Copia e incolla questo URL nel tuo browser:")
print(f"http://{alb_dns}")
print("=====================================================")
print("(Nota: Potrebbero volerci 2-3 minuti prima che le istanze scarichino Apache e superino gli Health Check dell'ALB)")

3 Esecuzione e Test

  1. Assicurati di aver sostituito i valori VPC_ID e le SUBNETS nel codice con i tuoi veri ID.
  2. Esegui lo script dal terminale: python deploy_alb_ec2.py.
  3. Lo script impiegherà circa 1-2 minuti. Al termine, ti restituirà un indirizzo web (il DNS dell'ALB).
  4. Apri il browser e vai a quell'indirizzo. Vedrai la pagina web di benvenuto.
  5. Prova l'High Availability: Ricarica la pagina più volte (potrebbe essere necessario forzare il refresh o usare la navigazione in incognito). Vedrai che il nome dell'istanza (hostname) cambierà, dimostrando che l'ALB sta bilanciando il traffico tra i due server!

4 Pulizia delle Risorse

Un'architettura complessa richiede una pulizia attenta per evitare costi. Poiché le risorse sono interdipendenti (non puoi eliminare un SG se è in uso da un'istanza), ti consigliamo di fare la pulizia tramite la Console AWS in questo ordine:

  1. Vai su EC2 > Load Balancers: Seleziona Lab-ALB-Web e clicca su Delete.
  2. Vai su EC2 > Target Groups: Seleziona Lab-Target-Group e clicca su Delete.
  3. Vai su EC2 > Instances: Seleziona le due istanze WebServer-Lab, clicca su Instance State e poi su Terminate instance.
  4. Attendi che le istanze siano completamente nello stato "Terminated".
  5. Vai su EC2 > Security Groups: Elimina prima EC2-SG-Lab e infine ALB-SG-Lab.

Hai appena creato un'infrastruttura web scalabile e sicura nel Cloud! ☁️
