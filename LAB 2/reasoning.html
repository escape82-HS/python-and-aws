LAB 2 – Client API & YAML in un’unica pagina HTML

Guida, codice e spiegazioni integrati

1. Cosa sta succedendo? Spiegazione guidata

▸ Obiettivo del laboratorio

Simuliamo il comportamento dello script Python client_http.py interamente nel browser:

  1. leggere un file YAML di configurazione;
  2. sovrascrivere alcuni valori con variabili d’ambiente (simulate);
  3. contattare un’API REST pubblica;
  4. mostrare i dati in una tabella HTML.

▸ Parsing YAML lato client (senza Python)

Usiamo la libreria js-yaml già presente nel CDN.

import * as yaml from 'js-yaml';        // equivale a "import yaml"
cfg = yaml.load(testoYAML);

yaml.load() restituisce oggetti/analoghi dizionari Python. Nel codice "vero" usiamo yaml.safe_load() per sicurezza; la versione JS lo fa già di default.

▸ Simuliamo le variabili d’ambiente

Il codice Python fa:

cfg["api"]["base_url"] = os.getenv("API_BASE_URL") or cfg["api"]["base_url"]

Nel browser non esiste os.getenv; perciò aggiungiamo due campi opzionali in fondo alla pagina (vedi sezione "Mock env") dove puoi scrivere temporaneamente l’URL base o il livello di log. Se li lasci vuoti, verrà usato il valore YAML caricato.

▸ Chiamata HTTP con fetch (alternativa a requests)

Python:

resp = requests.get(url)
resp.raise_for_status()
data = resp.json()

JavaScript:

const resp = await fetch(url);
if (!resp.ok) throw new Error(resp.status);
const data = await resp.json();

fetch è nativo nei browser; ritorna una Promise, quindi usiamo async/await.

▸ Creazione dinamica della tabella

La funzione buildTable(array) prende un array di oggetti (identico a quanto restituisce JSONPlaceholder).
1) Con Object.keys(arr[0]) ricava l’elenco delle colonne.
2) Con un doppio ciclo crea <th> e <td>.
3) Inserisce tutto nel DOM. Se l’API restituisce un altro schema, la tabella si adatta ugualmente.
